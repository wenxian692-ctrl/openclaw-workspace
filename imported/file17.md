# AI识别热量

### 1. 需求概述

当用户通过拍照或文字/语音方式提交饮食记录后，AI需根据信息清晰度自动识别并反馈结果。若信息足够明确，直接返回识别到的食物清单、重量、热量及三大营养素，并引导用户确认保存；若信息模糊或存在歧义，则启动多轮追问策略，逐步引导用户补充细节，直至达到可记录标准或满足终止条件。

### 2. 输入源与触发条件

#### 2.1 图片消息

用户通过以下任一方式触发图片发送：

*   **饮食任务“拍照打卡”**：在饮食任务弹层中点击“拍照打卡”，进入食物拍照页面后发送图片。
    
*   **底部快捷栏“AI识热量”按钮**：点击后直接进入食物拍照页面，拍摄或从相册选择图片后发送。
    
*   **对话框“📷”图标**：点击选择“AI识别热量”后，直接进入食物拍照页面，拍摄或从相册选择图片后发送。
    

#### 2.2 文字消息

用户通过以下任一方式触发文字发送：

*   **底部输入框手动输入**：在对话流底部输入框输入饮食描述，点击发送。
    
*   **语音输入**：点击输入框旁麦克风图标，通过语音转文字生成描述后发送。
    

#### 2.3 触发时机

AI接收到上述任一图片或文字消息后，自动进入饮食识别与追问流程。

### 3. 识别与响应逻辑

#### 3.1 图片识别

*   **技术能力**：调用图像识别服务，输出食物名称、置信度、可能重量范围及营养素估算。
    
*   **识别结果分类**：
    
    *   **明确识别（高置信度）**：单张图片中所有食物均被高置信度识别，且重量可合理估算（如通过参照物、常见分量）。
        
    *   **部分识别（中等置信度）**：部分食物被识别，但存在遮挡、模糊或混合菜肴难以拆分。
        
    *   **识别失败（低置信度）**：图片质量问题（光线过暗、滤镜、角度不佳）或图像中无清晰食物。
        

#### 3.2 文字识别

*   **技术能力**：调用NLP解析服务，提取食物名称、数量描述、烹饪方式等信息。
    
*   **识别结果分类**：
    
    *   **明确描述**：用户提供清晰的食物清单及量化信息（如“午餐：100g牛肉、一碗米饭、1个鸡蛋”）。
        
    *   **模糊描述**：用户仅提供大类（如“吃了炒菜和主食”）或遗漏关键信息。
        

*   **多餐次文字处理**：若用户一条消息中包含多个餐次的饮食描述（如“早餐吃了鸡蛋，午餐吃了牛肉”）
    
    *   **拆分原则**：根据用户描述中的餐次关键词（早餐/午餐/晚餐/加餐）进行拆分，生成多条独立记录。在对话流中**分多条消息依次展示**
        
    *   **自动保存**：每条记录生成后即自动保存至对应餐次的摄入数据中。

### 4. 追问策略

当识别结果分类为“部分识别/识别失败”或“模糊描述”时，AI启动多轮追问流程。

详细追问话术、分步逻辑及终止条件，详见独立文档详阅【[饮食记录追问策略](http://axure.sys.hxsapp.net/KHN5C5/#id=cakoef&p=%E9%A5%AE%E9%A3%9F%E8%AE%B0%E5%BD%95%E8%BF%BD%E9%97%AE&g=1)】说明

### 5. 多轮对话管理

*   追问过程需维护上下文状态，记录已确认的食物清单和待补全字段。
    
*   每一轮追问后，AI需根据用户新提供的信息更新临时记录，并判断是否满足记录条件。
    
*   支持用户随时打断或主动提供额外信息，AI应动态调整追问方向。
    

### 6. 记录确认与后续反馈

#### 6.1 自动保存机制

*   **触发条件**：当AI完成识别或追问，生成结构化的饮食记录（含食物清单、重量、热量、三大营养素）后，**系统自动将该记录写入用户今日摄入数据**，并实时更新能量看板。
    
*   **无需用户确认保存**：用户不需要点击任何按钮来触发“保存”动作。
    

#### 6.2 记录展示形态

AI在对话流中展示识别结果，格式参考：

#### 6.4 后续反馈联动

*   无论点击“确认无误”还是完成调整后，AI都会自动触发后续反馈话术，收集进食体验、饱腹感等信息，用于执行分和风险分计算。详阅【[饮食打卡反馈收集](http://axure.sys.hxsapp.net/KHN5C5/#id=o5qt1h&p=%E9%A5%AE%E9%A3%9F%E6%89%93%E5%8D%A1%E5%8F%8D%E9%A6%88&g=1)】说明
    

### 7. 异常处理

| 异常场景 | 处理方式 |
| --- | --- |
| 图片识别服务超时或不可用 | AI发送：“图片识别服务暂时不可用，请稍后重试或手动输入文字描述” |
| 文字解析服务失败 | AI发送：“我没理解您吃的是什么，能再描述一遍吗？比如‘米饭、红烧肉、炒青菜’” |
| 用户连续多次提供无效信息 | 根据终止条件停止追问，采用估算并提示偏差风险 |
| 用户要求取消记录 | 结束流程，不保存任何数据，返回对话界面 |

### 8. 边界条件

*   **同一餐次多次记录**：允许用户对同一餐次多次提交（如先拍照片，再补充文字），AI应合并信息或提示覆盖。
    
*   **敏感内容**：若检测到图片或文字包含违规内容，拒绝记录并提示。
