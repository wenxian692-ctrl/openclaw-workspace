# 任务推荐策略

### 1. 需求概述

用户完成一个任务打卡并收集反馈后，AI需要自主决策：**接下来推荐哪个任务最合适？** 这不是一个固定的规则引擎，而是考验AI对用户状态、时间窗口、任务优先级等多维度的综合判断能力。

**核心目标**：让用户感觉“AI很懂我”——推荐的时机对、任务对、话术对，自然而然地把全天任务串起来。

### 2. 触发时机

当以下条件全部满足时，AI触发任务推荐：

1.  **用户刚完成一个任务打卡**（饮食/运动/认知行为均可）
    
2.  **系统已收集完该任务的反馈**（如运动强度、饱腹感等）
    
3.  **当前还存在“进行中”且“未完成”的任务**
    

**一句话总结**：用户完成一个任务 → 收集反馈 → 为了不让用户“卡住”，只要还有没做的任务，AI主动推荐下一个最合适的任务。

### 3. AI决策目标（给AI的“北极星”）

AI在推荐下一个任务时，需要综合平衡以下三个目标：

| 目标 | 说明 |
| --- | --- |
| **时间合理性** | 优先推荐即将错过最佳执行窗口的任务，避免用户“来不及做” |
| **生理/行为合理性** | 考虑任务之间的自然衔接（如运动后拉伸、饭后散步） |
| **个体适配性** | 结合用户的历史偏好、当前反馈（如“挺累”则降低强度） |

**一句话给AI**：像一个了解用户的私人助理，在合适的时间，用合适的方式，提醒用户做最该做的事。

### 4. AI可参考的决策因素（非强制规则）

以下因素是AI做决策时的“输入信号”，**不作为硬性规则**，由AI自主判断权重和组合方式：

| 因素类别 | 具体信号 | 示例 |
| --- | --- | --- |
| **时间维度** | 任务的剩余可执行时间 | A任务还剩10分钟截止，B任务还有2小时 → 可能优先A |
| **任务类型** | 饮食/运动/认知行为 | 通常饮食优先，但用户刚吃完可能不急着推下一餐 |
| **用户反馈** | 上一任务的完成感受 | 用户说“极限了”，应避免推高强度运动 |
| **用户历史** | 用户过去在相同时段的偏好 | 用户习惯午饭后立即散步，可优先推散步任务 |
| **任务关联性** | 任务之间的自然衔接 | 运动后推拉伸、饮食后推记录体重 |
| **任务完成度** | 当前已完成/未完成情况 | 今天运动KPI还差一个，可优先推运动 |

### 5. 推荐呈现方式

AI确定推荐任务后，需要根据用户当前所在页面，选择合适的呈现方式：

| 用户所在位置 | 推荐方式 | 说明 |
| --- | --- | --- |
| **今日任务卡页面** | 直接在卡片上**高亮推荐任务** | 加边框、呼吸灯、或置顶显示，让用户一眼看到 |
| **对话流页面** | AI主动发送一条推荐消息 + 快捷按钮 | 话术自然，按钮可直接跳转任务 |

**推荐话术示例（仅供参考，AI可自主生成）**：

> “刚完成快走，状态不错！现在可以开始核心训练了，平板支撑 3组×30秒，要现在开始吗？”  按钮：【开始训练】

### 6. 边界条件与处理建议

| 场景 | AI可考虑的处理方向 |
| --- | --- |
| 无进行中任务 | 发送鼓励语：“今天任务都完成啦，太棒了！好好休息～” |
| 用户多次忽略推荐 | 降低推荐频率，或询问用户“是不是不想被提醒？” |
| 推荐任务因时间已过无法执行 | 自动顺延到下一个最合适的任务，并向用户说明 |
| 用户反馈负面（如“太累了”） | 后续推荐降低强度，或推恢复性任务（如拉伸、休息提醒） |

### 7. 数据记录与迭代

为了让AI持续进化，需要记录以下数据供后续分析和训练：

| 数据项 | 用途 |
| --- | --- |
| 每次推荐的决策因子（AI可输出的思考过程） | 分析AI的决策逻辑是否合理 |
| 用户是否接受推荐（点击/忽略/拒绝） | 评估推荐准确率 |
| 用户对推荐任务的反馈 | 判断推荐是否真正符合用户需求 |
| 推荐到任务完成的转化率 | 衡量推荐效果的核心指标 |

### 8. 设计说明（给AI的“人设”）

*   **像朋友一样**：推荐不是命令，是提醒和关心，用户拒绝也没关系
    
*   **有温度**：结合上一任务的反馈，话术要有共情感（如“刚才辛苦了，接下来这个比较轻松”）
    
*   **不刷屏**：避免短时间内多次推荐，给用户消化时间
    
*   **可解释**：如果用户问“为什么推这个”，AI能给出简单理由（如“因为快到截止时间了”）
    

### 9. 成功标准（供验收参考）

| 维度 | 理想状态 |
| --- | --- |
| **推荐时机** | 用户不觉得被打扰，反而觉得“刚好想到” |
| **推荐内容** | 用户接受推荐的意愿高，转化率逐步提升 |
| **话术自然度** | 用户能感受到是“小享”在说话，而非系统通知 |
| **个性化** | 不同用户、不同状态下的推荐有明显差异 |
