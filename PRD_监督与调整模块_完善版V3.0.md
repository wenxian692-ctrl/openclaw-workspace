## 1. 文档目的

本文档定义好享健康 APP 中"监督与调整"模块的完整产品方案，供产品、前端、后端、AI、数据、测试、顾问团队评审与落地。

本模块不是单纯的提醒系统，而是 AI Agent 驱动健康管理的控制层，负责在全生命周期内持续感知用户状态，识别信号，生成动作，并验证动作是否生效。

### 1.1 覆盖阶段

| 阶段 | 说明 |
|------|------|
| 首次测评 | 用户初次使用时的测评流程 |
| 报告后继续获取方案 | 测评报告生成后的方案获取 |
| 方案生成与待激活 | 方案确认与激活 |
| 付费待支付、待发货、待签收 | 订单履约阶段 |
| Day0 与 Day1 | 服务启动阶段 |
| 28 天服务执行期 | 方案执行阶段 |
| Day7/14/21 周复评 | 周期性复评 |
| Day28 阶段复评 | 阶段总结复评 |
| 下阶段承接 | 阶段切换 |
| 服务终止与退款评估 | 服务结束 |

## 4. 核心设计原则

### 4.1 全生命周期持续监督

监督不是只发生在执行期，而是贯穿以下所有阶段：测评前和测评中、方案待获取与待激活、履约等待期、Day0和Day1、执行期、复评期、阶段切换期、终止和退款评估期。

### 4.2 显性与隐性并行

监督必须同时覆盖两类信息：显性监督（用户明确完成、反馈、表达）和隐性监督（不打卡、不回复、临近截止才完成、打开不执行、短回复、重复同一困难）。

### 4.3 规则、AI、混合判定分层

规则引擎负责明确阈值、明确时点、必须稳定的判断；AI负责自然语言理解、阻碍识别、语义归因；混合判定负责AI结构化提取后由规则引擎落动作。

### 4.4 风险优先级最高

任何阶段都遵循优先级：医疗和安全边界 > 正在进行中的测评和复评 > 当前主任务闭环 > 风险纠偏 > 日常主动问询 > 商业转化和续费。

### 4.5 调整必须回写和验证

系统必须记录调整原因、调整内容、用户是否接受、调整后是否改善、是否继续或恢复或升级或终止。

## 5. 决策思维链

监督与调整模块的决策必须按以下8步执行：

| 步骤 | 输入 | 输出 |
|------|------|------|
| Step 1: 识别当前所处阶段 | 用户状态数据 | 当前阶段判定 |
| Step 2: 收集监督事件 | 显性事件+隐性事件 | 事件列表 |
| Step 3: 生成信号 | 事件列表 | 信号列表 |
| Step 4: 更新分数 | 信号+历史分数 | 新分数 |
| Step 5: 判断生命周期 | 分数+趋势+目标 | 生命周期 |
| Step 6: 判定干预等级 | 分数+信号 | 干预等级 |
| Step 7: 选择动作和触达渠道 | 干预等级+用户偏好 | 动作+渠道 |
| Step 8: 回写与效果验证 | 动作执行结果 | 效果评估 |

## 6. 判定架构

### 6.1 规则引擎直接判定

以下场景由规则引擎直接判定：任务是否逾期、连续几天未打卡、测评/复评/方案确认是否超时、Day7/14/21/28是否到点、体重是否缺失到阈值、体重是否反弹、是否达到阶段目标、是否触发Push、是否触发L3人工、生命周期阈值。

### 6.2 AI语义判定

以下场景由AI负责识别：负向情绪、信任危机、求快心理、高危红线表达、用户主诉的阻碍类型、是否真实执行、当前行为任务是否适配、外食/夜宵/出差等场景。

### 6.3 混合判定

以下场景采用混合判定：行为评分、平台期归因、行为替换建议、行为标签状态流转、任务难度调整参数、是否继续当前行为任务。

## 8. 生命周期定义

### 8.1 生命周期总览

| 阶段 | 名称 | 条件 | 重点 |
|------|------|------|------|
| S1 | 新手启动期 | 注册后到方案激活后前7天 | 补档、Day0/Day1引导、降低摩擦 |
| S2 | 稳定执行期 | 执行分≥65，风险分<30 | 稳定执行、适度升级，强化反馈 |
| S3 | 平台期风险 | 执行分≥60，真实平台期 | 平台期解释、饮食微调，防流失 |
| S4 | 风险期 | 风险分≥50或执行分<45 | 拉回、缩包、Plan B、必要人工 |
| S5 | 稳态维持期 | 达到阶段目标且稳定7-14天 | 频率下调、防反弹、维持习惯 |

## 9. 监督来源体系

### 9.1 显性监督来源

| 类别 | 事件 |
|------|------|
| 测评与复评 | 初评开始/中断/完成、周复评开始/完成、阶段复评开始/完成、报告生成/确认、方案确认 |
| 执行期 | 任务完成/跳过/补录、追问回复/拒绝、任务质量低 |
| 饮食 | 饮食记录完成、AI识别热量完成、饱腹感/进食速度反馈、偷吃/暴食/外食/夜宵主诉 |
| 运动 | 运动打卡完成、强度/吃力反馈 |
| 行为 | 数据追踪任务完成、非数据追踪任务Day1/Day2/Day3、行为追问回复、用户正负反馈 |
| 生理 | 用户主动记录体重、体脂秤同步、腰围/臀围记录、睡眠同步、生理期记录 |
| 履约 | 发货、物流状态更新、签收回调、断供预警 |
| 交互 | 负向情绪/求快/信任危机/高危红线表达、用户主动求助 |

## 11. 分数规则详解

### 11.1 执行分（execution_score）

#### 11.1.1 加分规则

| 行为 | 加分 | 说明 |
|------|------|------|
| 当日主任务按时完成 | +2/项 | 每日 |
| 连续3天完成今日三件事 | +5 | 周期奖励 |
| 连续7天完成率≥80% | +8 | 周期奖励 |
| 主动补录有效数据 | +1 | 主动行为 |
| 有效回复主动问询 | +1 | 互动奖励 |
| 完成当周复评 | +8 | 里程碑 |

#### 11.1.2 扣分规则

| 行为 | 扣分 | 说明 |
|------|------|------|
| 单项任务逾期 | -2/项 | 超时 |
| 单项任务未完成 | -4/项 | 未做 |
| 连续2天核心任务未完成 | -6 | 连续 |
| 连续3天核心任务未完成 | -12 | 严重 |
| 追问已触发但无回复 | -2 | 忽视 |
| 低质量打卡 | -2 | 敷衍 |

#### 11.1.3 使用原则

平台期不直接扣执行分；用户主动说明客观困难时，执行分扣减减半；投诉和医疗边界不走执行分。

### 11.2 能量分（energy_score）

#### 11.2.1 加分规则

| 行为 | 加分 | 说明 |
|------|------|------|
| 睡眠≥7小时 | +5 | 充足睡眠 |
| 疲劳自评较好 | +4 | 主观感受 |
| 运动反馈"轻松/刚好" | +3 | 适中运动 |
| 用户主动反馈状态恢复 | +3 | 恢复信号 |

#### 11.2.2 扣分规则

| 行为 | 扣分 | 说明 |
|------|------|------|
| 睡眠<5小时 | -10 | 严重不足 |
| 失眠主诉 | -8 | 睡眠问题 |
| 运动反馈"很累/极限" | -5 | 过度运动 |
| 连续熬夜 | -8 | 作息问题 |
| 生理期影响 | -4 | 生理波动 |
| 出差或场景波动 | -4 | 环境变化 |
| 明显负向情绪 | -4 | 心理状态 |

### 11.3 风险分（risk_score）

#### 11.3.1 加分规则

| 行为 | 加分 | 说明 |
|------|------|------|
| 连续2天无主任务完成 | +15 | 轻度风险 |
| 连续3天无主任务完成 | +25 | 中度风险 |
| 连续3天无体重数据 | +15 | 数据缺失 |
| 连续7天无体重数据 | +30 | 严重缺失 |
| 7天均重反弹≥0.5kg | +20 | 反弹 |
| 投诉或退费语义 | +50 | 高风险 |
| 信任危机 | +20 | 流失风险 |
| 高危红线 | +100 | 最高风险 |
| 医疗边界 | +100 | 安全边界 |

#### 11.3.2 衰减规则

每日无新增风险事件时衰减10%，最低衰减到0分。

#### 11.3.3 使用原则

risk_score≥80直接L3；高危红线和医疗边界不等待二次确认。

## 12. 干预等级与动作类型

### 12.1 L0 不干预

适用于风险低、执行稳定、当前无必要额外触达的场景。

### 12.2 L1 提醒

适用于单次未完成、单次数据缺失、轻度拖延、初次复评超时。动作包括首页卡片、站内消息、AI轻提醒、Push。

### 12.3 L2 调整

适用于连续偏差、低能量、平台期、多次提醒无效、行为任务明显不适配。动作包括缩小任务包、任务降级、替代任务、行为替换、饮食方法微调、周末模式、Plan B。

### 12.4 L3 人工

适用于高危红线、医疗边界、投诉退费、连续多次L2无效、高价值用户长期沉默。动作包括顾问接管、工单、电话/企微、医学或营养师介入。

## 13. 全生命周期分阶段 PRD

### 13.1 阶段A：首次测评

覆盖状态：NOT_STARTED、IN_PROGRESS、INTERRUPTED、COMPLETED、EXPIRED、CANCELED。

监督重点：是否进入首次测评、是否中断、是否长时间停留不完成、是否出现非服务适配风险。

核心信号：assessment_resistance、assessment_timeout、service_ineligible_risk。

核心动作：首次中断-首页恢复卡+AI轻召回；48小时内第二次中断-缩短入口说明；72小时仍未继续-降低主动催促频率；高风险或不可服务-终止当前测评并解释边界。

### 13.2 阶段B：报告已出待继续获取方案

监督重点：用户是否理解报告、是否愿意继续获取方案、是否被成本/难度/信任问题阻断。

核心信号：plan_hesitation、trust_crisis、urgency_bias。

核心动作：优先解释报告不先催转化；用AI解答"为什么需要继续获取方案"；用首页卡或站内卡承接。

### 13.3 阶段C：方案前阶段测评P1

监督重点：是否完成方案前补充信息、是否中断、是否因问题复杂产生疲劳。

### 13.4 阶段D：方案已出未激活或付费未支付

覆盖状态：READY、AWAITING_PAYMENT。

监督重点：用户是否确认方案、用户是否激活免费方案、用户是否发起支付但未完成。

核心动作：免费方案-优先解释后促激活；付费方案-解释权益流程限制不夸大承诺；支付发起未完成-站内卡优先必要时Push一日最多1次。

### 13.5 阶段E：付费后待发货待签收

覆盖状态：AWAITING_DELIVERY、AWAITING_SIGN。

核心规则：发货和签收以实时回调为主，兜底链路定时对账防回调丢失。

核心动作：发货成功-即时Push；签收成功-即时Push+Day1预告；回调异常-优先对账不先骚扰用户。

### 13.6 阶段F：激活后待Day0

监督重点：用户是否完成Day0准备、是否具备执行条件。

核心动作：Day0只强调准备不强执行评价；截止前做轻提醒；当天未做完次日首页优先承接。

### 13.7 阶段G：执行期Day1-Day6

监督重点：今日三件事是否执行、饮食/运动/行为三类任务是否闭环、是否出现掉队/低能量/外食/夜宵/失眠/出差等场景。

核心动作：正常-L0或轻反馈；单次未完成-L1；连续未完成-L2；高风险-L3。

### 13.8 阶段H：Day7/14/21周复评

监督重点：用户是否点击复评入口、是否完成复评、目标进度是否达标、未达标原因是什么。

核心规则：08:00出口；当日20:00未开始-Push；+24h未开始-再次提醒；+48h未完成-进入超时逻辑。

周复评后可做：饮食方法微调、任务参数微调、行为优先级调整、运动强度轻调。周复评后不做：服务阶段切换、下一阶段方案生成。

### 13.9 阶段I：Day28阶段复评

监督重点：用户是否进入阶段复评、是否达到当前阶段目标、未达标原因是什么。

核心规则：08:00出口；当日20:00未开始-Push；+24h未开始-Push+首页强提醒；+48h未完成-升级L2。

达标处理：当前阶段完成、进入下一服务阶段、生成下一阶段方案。

未达标处理：未达标不直接进入下一阶段必须先归因。归因类型包括A1执行不足、A2饮食方法不匹配、A3能量或生理波动、A4场景干扰过强、A5产品或履约问题。

### 13.10 阶段J：终止与退款评估

覆盖状态：TERMINATED。

终止原因：Day28未达标归因进入终局终止、提前终止规则命中、高风险或投诉导致服务终止。

终止后仍需监督：退款流程进度、投诉闭环、人工跟进是否完成、用户是否点击重新开始。

#### 14.4.1 打卡类型定义

| 类型 | 执行周期 | 说明 |
|------|----------|------|
| 数据追踪类 | 1天 | 获取具体数据，如晨起体重、喝水量等 |
| 行为习惯类 | 3天 | 形成健康习惯，如先吃蔬菜、饭后站立等 |

#### 14.4.2 打卡状态

| 状态 | 定义 |
|------|------|
| 待打卡 | 今日待完成 |
| 已打卡 | 今日已完成 |
| 漏打卡 | 昨日未完成（仅Day2/Day3） |

#### 14.4.3 打卡方式

数据类任务在首页点击打卡按钮即可完成，无需打开二级界面。行为习惯类任务需拉起二级打卡页面，展示Day1/Day2/Day3连续打卡进度。

#### 14.4.4 反馈收集-四维度评分

行为习惯类任务完成后触发AI追问，收集四个维度评分：

| 维度 | 权重 | 说明 |
|------|------|------|
| 真实性 | 30% | 用户是否真实执行 |
| 执行质量 | 25% | 执行的程度如何 |
| 效果感知 | 20% | 用户是否感受到效果 |
| 可持续性 | 25% | 是否容易坚持 |

**评分公式：**

```
behavior_score = 0.30*真实性 + 0.25*执行质量 + 0.20*效果感知 + 0.25*可持续性
```

#### 14.4.6 Day1/Day2/Day3目标

| 天数 | 目标 | 获取维度 |
|------|------|----------|

| Day3 | 判断是否可形成习惯 | 可持续性 |

|------|----------|----------|

| 追问无回复 | 连续2次追问无回应 | L2 |
| 用户反馈困难 | 明确说难/没时间/不想做 | L2 |

#### 14.4.8 3天评估动作矩阵

| 完成情况 | 评分 | 动作 |
|----------|------|------|
| 0/3或1/3完成，无回复 | - | 隐性掉队，标签保留，降入P3 |
| 0/3或1/3完成，用户明确说难 | 核心动作合理 | 先难度调整 |
| 0/3或1/3完成，用户明确说难 | 核心动作不合理 | 行为替换 |
| 0/3或1/3完成，用户明确拒绝 | - | 任务移除 |

| 3天完成 | ≥4.2 | 继续巩固/流转下一任务，标签weakened |
| 3天完成 | 3.4-4.2 | 继续当前任务，不升级难度 |
| 3天完成 | 2.6-3.4 | 优先难度调整 |
| 3天完成 | 1.8-2.6 | 优先行为替换，新任务进P1顶部 |
| 3天完成 | <1.8 | 任务移除，次日切换下一任务，标签保留 |

#### 14.4.9 应用维度分数映射

行为干预任务调整除行为评分驱动外，还受应用维度分数影响。

**能量分对行为干预的影响：**

| 能量分范围 | 行为任务调整策略 |
|------------|------------------|
| ≥80 | 可提升行为任务难度，维持正常节奏 |
| 65-79 | 维持当前行为任务难度 |
| 50-64 | 降低行为任务频次/时长 |
| 35-49 | 替换为更轻松的行为任务 |
| <35 | 暂停行为干预任务，优先恢复能量 |

**执行分对行为干预的影响：**

| 执行分范围 | 行为任务调整策略 |
|------------|------------------|
| ≥70 | 任务完成良好，可升级挑战 |
| 55-69 | 维持当前任务难度 |
| 45-54 | 降低任务难度，缩小任务包 |
| 30-44 | 简化任务或替换为更易执行任务 |
| <30 | 暂停行为干预，分析掉队原因 |

**风险分对行为干预的影响：**

| 风险分范围 | 行为任务调整策略 |
|------------|------------------|
| <15 | 正常执行行为任务 |
| 15-29 | 关注行为完成情况，轻度干预 |
| 30-49 | 降低行为任务要求，优先完成 |
| 50-79 | 简化行为任务，缩短周期 |
| ≥80 | 暂停行为干预，升级L3人工介入 |

#### 14.4.10 调整动作类型

| 动作 | 说明 |
|------|------|
| 继续当前任务 | 保持现有任务不变 |
| 难度调整 | 参数调整（次数/频次/时长/执行窗口） |
| 行为替换 | AI生成更适配的新任务 |
| 任务移除 | 移除当前任务 |
| 完成并流转 | 进入下一任务 |

## 15. 日常问询监督与调整规则

### 15.1 问询类型

| 类型 | 说明 |
|------|------|
| 补齐初始问卷缺失数据 | 针对初始问卷中应采集但用户未填写的数据字段 |
| 行为习惯风险筛查 | 通过行为习惯库中的筛查问题识别高风险行为 |
| 营养师建议的日常采集 | 将营养师出测评报告所需了解的个人健康数据分散采集 |

### 15.2 触发条件

#### 15.2.1 前置条件

用户当前无更高优先级事项（无进行中测评/打卡任务），且pending_questions非空，距上次日常问询>3小时。

#### 15.2.2 触发场景

| 场景 | 触发条件 |
|------|----------|
| 用户打开APP时 | 待回答问题池非空、初始问卷缺失字段、行为习惯库待筛查问题 |
| 用户主动聊天 | AI可在对话中自然插入一个数据采集问题 |

### 15.3 问询规则

#### 15.3.1 单次问题数量

每次主动问询最多1个问题。

#### 15.3.2 问题形式

提问形式优先采用：选择题、数值题、评分题。

#### 15.3.3 每日限制

每次主动询问间隔需要≥3h，每日最多询问次数≤3。

#### 15.3.4 话术要求

避免问卷化表达，采用自然语言改写。例如原始问题"睡眠情况"，AI提问"我 想了解一下你的作息情况，你平时大概每天睡几个小时呢？"

### 15.4 用户回答状态识别

| 状态 | 定义 | 后续策略 |
|------|------|----------|
| answered | 用户提供有效回答 | 数据入库，一次性问题不再询问；周期性问题冷却7天后再询问 |
| skip | 用户未提供答案但未拒绝 | 不入库，冷却7天后再次询问 |
| refuse | 用户明确拒绝回答 | 不入库，问题标记拒绝，冷却14天内不再询问 |
| timeout | 用户超5分钟未回复 | 不入库，3天后可再次询问 |
| unrecognized | 回复无法识别 | AI澄清一次，仍无法识别则问题返回数据池 |
| topic_shift | 用户转移话题 | 不入库，问题返回数据池，冷却7天后再次询问 |

### 15.5 监督信号

| 信号 | 触发条件 | 干预等级 |
|------|----------|----------|
| 问询无响应 | 连续2次问询无有效回复 | L1 |
| 用户拒绝 | 明确拒绝回答相关问题 | L1 |
| 用户拖延 | 周期性问询延迟超过7天 | L2 |

### 15.6 调整动作

| 场景 | 调整动作 |
|------|----------|
| 问询频率过高 | 降低每日问询次数 |
| 用户抗拒 | 延长冷却时间，减少问题难度 |
| 周期性问询效果差 | 调整问询时机，选择用户活跃时段 |

## 16. 方案调整规则

### 16.1 周期内微调

28天服务周期内允许的主要调整包括饮食方法、饮食任务参数、运动任务难度/频次/时长、行为任务难度和顺序、提醒频率。不直接调整服务阶段本身、不因单次波动切换下一阶段。

### 16.2 周复评后调整

周复评后可做饮食方法微调、任务参数微调、行为优先级调整、运动强度轻调。不做服务阶段切换、下一阶段方案生成。

### 16.3 阶段复评后调整

达标：当前阶段完成、进入下一阶段、生成下一阶段方案。

未达标：未达标不直接进入下一阶段必须先归因。归因类型包括A1执行不足、A2饮食方法不匹配、A3能量或生理波动、A4场景干扰过强、A5产品或履约问题。

| suspected | 疑似 |
| confirmed | 确认 |
| intervening | 干预中 |
| weakened | 弱化 |
| resolved | 已解决 |

### 17.2 标签进入条件

进入条件：测评命中、主动问询承认、AI语义识别明确主诉、隐性信号重复出现≥2。

### 17.3 标签升级条件

升级条件：7天内同类问题重复出现、当前行为任务失败、隐性和显性反馈同时命中。

### 17.4 标签弱化条件

满足任一即可：连续1个行为周期behavior_score≥4.2且7天内无重复负向信号。

### 17.5 标签解决条件

满足以下组合：连续2个相关任务周期表现良好，或连续14天无相关显性和隐性信号，若有客观数据不恶化。

### 17.6 标签保留条件

以下情况标签保留在confirmed或intervening：任务未完成、行为评分低、用户明确拒绝、隐性负向信号仍持续。

## 21. 验收标准

### 21.1 监督有效

系统能识别显性与隐性监督事件；体重、签收等关键事件实时优先；定时检测只做缺失和兜底。

### 21.2 判定可解释

能说清楚某次动作为什么触发；能回放event→signal→score→action。

### 21.3 动作可执行

任务调整能落到实际任务；行为替换能生成新任务并进入P1；终止和退款评估有明确入口。

### 21.4 渠道不打扰

Push有冷却和优先级；高风险优先，普通提醒克制。

### 21.5 全生命周期打通

首次测评、方案、执行、复评、阶段流转、终止全部被覆盖。
