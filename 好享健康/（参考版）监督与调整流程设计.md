## 一、设计总则

### 1.1 核心目标

| 目标 | 说明 |
| --- | --- |
| **全周期感知** | 覆盖测评期、执行期、复评期、维持期全生命周期信号 |
| **多源信号融合** | 体脂秤、任务系统、对话内容、行为埋点多源数据融合 |
| **自动决策** | 基于分数和规则，自动判定干预等级（L1/L2/L3） |
| **动态调整** | 根据执行效果，调整饮食/运动/行为干预策略 |
| **生命周期管理** | 实时更新用户状态（S1~S5），触发对应运营策略 |

## 二、分数体系与计算规则

### 2.1 三大核心分数

| 分数 | 含义 | 初始值 | 更新频率 | 计算要素 |
| --- | --- | --- | --- | --- |
| **执行分** | 任务完成质量与依从性 | 60 | 每日 | 任务打卡、超时、连续完成、饮食/运动/行为反馈 |
| **能量分** | 用户身心承受力 | 70 | 每日 | 睡眠、疲劳自评、运动反馈、情绪 |
| **风险分** | 安全红线与流失概率 | 0 | 实时/每日 | 连续未打卡、体重反弹、医疗边界、投诉词、饮食/运动异常 |

### 2.2 执行分计算规则

```markdown
## 计算公式

执行分 = 初始值 (60) + 加分项 - 扣分项

## 加分项
| 行为 | 加分 | 说明 |
|------|------|------|
| 任务按时完成 | +2/个 | 每日最多 3 个任务 |
| 任务提前完成 | +1/个 | 在推荐窗口期内完成 |
| 连续 3 天全完成 | +5 | 额外奖励 |
| 连续 7 天全完成 | +10 | 额外奖励 |
| 主动补录 | +1/个 | 超时后主动补打卡 |
| 进食速度正常/慢慢吃 | +1/餐 | 饮食反馈收集 |
| 饱腹感"刚好" | +1/餐 | 饮食反馈收集 |
| 运动反馈"轻松/刚好" | +1/次 | 运动反馈收集 |
| 行为打卡连续 3 天 | +5 | 行为任务连续完成 |
| 行为打卡连续 7 天 | +10 | 行为任务连续完成 |
| 日常问询响应 | +1/次 | 用户回答日常问询 |
| 复评完成 | +10/次 | 完成周复评/阶段复评 |

## 扣分项
| 行为 | 扣分 | 说明 |
|------|------|------|
| 任务超时完成 | -1/个 | 在补录期内完成 |
| 任务未完成 | -5/个 | 超过最晚截止时间 |
| 连续 3 天未完成 | -10 | 额外惩罚 |
| 连续 7 天未完成 | -20 | 额外惩罚 |
| 任务质量差 | -2/个 | AI 识别敷衍打卡 |
| 进食过快（<15 分钟） | -1/餐 | 饮食反馈收集 |
| 饱腹感"很撑"或"还能吃" | -1/餐 | 饮食反馈收集 |
| 运动反馈"挺累/极限了" | -1/次 | 运动反馈收集 |
| 行为打卡中断 | -3/次 | 行为任务中断 |
| 复评未完成 | -15/次 | 周复评/阶段复评未完成 |

## 边界
- 上限：100 分
- 下限：0 分

```

### 2.3 能量分计算规则

```markdown
## 计算公式

能量分 = 初始值 (70) + 加分项 - 扣分项

## 加分项
| 行为/状态 | 加分 | 数据来源 |
|-----------|------|----------|
| 睡眠≥7h | +5 | 用户上报/设备同步 |
| 疲劳自评≤2 | +5 | 日常问询 |
| 运动后反馈积极 | +3 | 打卡后反馈 |
| 对话情绪积极 | +2 | AI 情绪分析 |
| 体重下降符合预期 | +5 | 体重记录 |
| 主动反馈困难 | +3 | 用户主动说明情况 |

## 扣分项
| 行为/状态 | 扣分 | 数据来源 |
|-----------|------|----------|
| 睡眠<5h | -10 | 用户上报/设备同步 |
| 疲劳自评≥4 | -5 | 日常问询 |
| 运动后反馈消极 | -3 | 打卡后反馈 |
| 对话情绪消极 | -2 | AI 情绪分析 |
| 体重反弹>0.5kg | -5 | 体重记录 |
| 生理期（女性） | -5 | 用户上报 |
| 出差/旅行 | -5 | 用户主动反馈 |
| 失眠 | -8 | 用户主动反馈 |

## 边界
- 上限：100 分
- 下限：0 分

```

### 2.4 风险分计算规则

```markdown
## 计算公式

风险分 = 触发事件加分（取最大值，不累加） - 衰减

## 触发事件加分
| 事件 | 加分 | 说明 |
|------|------|------|
| 连续未打卡 1 天 | +10 | 每日累加 |
| 连续未打卡 3 天 | +30 | 触发 L1 干预 |
| 连续未打卡 7 天 | +60 | 触发 L3 干预 |
| 体重反弹 0.1kg | +5 | 每次反弹累加 |
| 体重反弹>1kg | +20 | 单次触发 |
| 医疗边界词 | +100 | 立即触发 L3 人工 |
| 极端减重行为 | +50 | 如绝食、过度运动 |
| 投诉词 | +50 | "没用""骗人""退款" |
| 连续 3 餐"很撑" | +20 | 过量摄入风险 |
| 连续 5 餐"还能吃" | +15 | 摄入不足风险 |
| 连续 3 次运动"极限了" | +25 | 过度训练风险 |
| 连续 7 天无体重数据 | +35 | 逃避心态，流失前兆 |
| 用户反馈偷吃/暴食 | +15 | 单次事件 |
| 用户反馈失眠 | +10 | 单次事件 |
| 用户反馈出差/无法打卡 | +10 | 单次事件 |

## 衰减规则
- 每日无新事件时，风险分衰减 10%（向下取整）
- 最低衰减到 0

## 边界
- 上限：100 分
- 下限：0 分

```

## 三、生命周期更新规则

### 3.1 生命周期状态定义

| 状态 | 名称 | 定义 | 典型特征 |
| --- | --- | --- | --- |
| **S1** | 新手启动期 | 注册 0-7 天，数据空白 | 执行分<60，需引导 |
| **S2** | 稳定执行期 | 执行分≥80，体重下降 | 依从性高，正向反馈 |
| **S3** | 平台期风险 | 体重连续 7 天无变化（±0.2kg），执行分≥70 | 努力但无效，易挫败 |
| **S4** | 风险期 | 连续 3 天未打卡/执行分<50/体重上升/风险分≥50 | 掉队、反弹、投诉风险 |
| **S5** | 稳态维持期 | 达成阶段目标，维持 2 周 | 执行分≥80，体重稳定 |

### 3.2 生命周期流转规则

```markdown
## S1 → S2
条件：执行分≥80 且 体重 7 天均值下降
动作：进入稳定执行期，任务难度可适度提升

## S2 → S3
条件：体重连续 7 天无变化（±0.2kg）且 执行分≥70
动作：触发平台期干预，调整饮食/运动方案

## S2 → S4
条件：连续 3 天未打卡 OR 执行分<50 OR 体重 7 天均值上升 OR 风险分≥50
动作：触发风险干预，L1/L2/L3 升级

## S3 → S2
条件：体重开始下降（7 天均值下降>0.5kg）
动作：退出平台期，恢复常规策略

## S3 → S4
条件：执行分<50 OR 风险分≥60
动作：平台期 + 风险期叠加，优先处理风险

## S4 → S2
条件：执行分≥70 且 风险分<30 且 体重开始下降
动作：风险解除，恢复稳定执行

## S2/S3/S4 → S5
条件：达成阶段目标 且 维持 2 周（执行分≥80，体重稳定±0.2kg）
动作：进入维持期，降低任务频率，强化习惯

## S5 → S2
条件：新周期开始（用户选择继续减重）
动作：重新进入执行期，目标更新

```

## S5 稳态维持期
- 策略：习惯固化，预防反弹
- 任务：降低频率，强化核心习惯
- 干预：L1 为主，月度复评

## 九、异常处理与降级方案

### 9.1 异常场景处理

```markdown
## 场景 1：AI 服务不可用
- 降级：L1 干预改用固定话术模板
- 恢复：AI 服务恢复后自动切换回动态话术

## 场景 2：大数据计算延迟
- 降级：使用昨日分数，延迟干预决策
- 恢复：计算完成后补发干预

## 场景 3：事件总线积压
- 降级：优先处理高风险事件（风险分≥80）
- 恢复：积压清理后正常处理

## 场景 4：用户设备离线
- 降级：Push 改为短信，保留干预记录
- 恢复：用户上线后同步状态

## 场景 5：饮食反馈连续拒绝
- 降级：暂停触发 3 天，避免用户反感
- 恢复：3 天后重新尝试触发

## 场景 6：运动反馈用户不回应
- 降级：等待 2 分钟后结束，标记为"待补充"
- 恢复：下次打卡时重新尝试

## 场景 7：行为打卡追问用户打断
- 降级：尊重用户意愿，结束追问
- 恢复：下次打卡时正常触发

## 场景 8：日常问询用户连续拒绝
- 降级：该问题冷却 14 天，暂停触发
- 恢复：14 天后重新尝试

## 场景 9：主聊监督用户反馈异常
- 降级：记录反馈，暂不触发调整，等待人工审核
- 恢复：AI 服务恢复后补发调整建议

## 场景 10：平台期识别误判
- 降级：优先按"假努力"处理，督促执行
- 恢复：连续 14 天数据确认后触发平台期干预

```
